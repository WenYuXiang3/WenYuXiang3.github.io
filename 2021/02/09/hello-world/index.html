Basic Syntax of Markdown

2021-02-09

Markdown is a markup language in plain text format. Through simple markup grammar, it can make ordinary text content have a certain format.

Advantages

  1. Because Markdown is pure text, the same editing effect can be obtained as long as it supports Markdown, which can free the author from the trouble of typesetting and concentrate on writing.
  2. Simple operation. For example: mark a title when editing WYSIWYG, first select the content, and then click the title button in the navigation bar to select several levels of titles. There are three steps. Markdown only needs to add # before the title content.

Disadvantages

  1. Need to memorize some grammar (of course, it is very simple. Learn in five minutes).
  2. Some platforms do not support Markdown editing mode.

1, Title

Add # in front of the text you want to set as the title to indicate
One # is the first-level heading, the second # is the second-level heading, and so on. Support six levels of headings.

Note: The standard grammar is generally written after # followed by a space.

2, Font

Bold

Wrap the text to be bolded with two ‘*’

Italic

Wrap the left and right characters with an ‘*’

Italic Bold

Wrap the slanted and bold text with three ‘*’

Strikethrough

Wrap the text to be struck out with two ’~~‘

Example:

The effect is as follows:

This is bold text
This is slanted text
This is italic and bold text
This is the strikethrough text

3, Reference

Just add > before the quoted text. References can also be nested, such as adding two >> three >>>nth…
